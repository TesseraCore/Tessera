## üêõ Reporting Issues

### Before Reporting

1. **Search existing issues** - Your issue may already be reported
2. **Check documentation** - The answer might be in the docs
3. **Try the latest version** - Update to the latest release

### How to Report

Use [GitHub Issues](https://github.com/tessera/tessera/issues) for:

- **Bug Reports**: Something isn't working as expected
- **Feature Requests**: Suggest new functionality
- **Documentation Issues**: Documentation errors or improvements

### Bug Report Template

When reporting a bug, please include:

```markdown
**Description**
A clear description of the bug.

**Steps to Reproduce**
1. Step one
2. Step two
3. See error

**Expected Behavior**
What should happen.

**Actual Behavior**
What actually happens.

**Environment**
- Tessera Version: [e.g., 0.1.0]
- Browser: [e.g., Chrome 120]
- OS: [e.g., macOS 14.0]
- Node.js Version: [e.g., 20.0.0]

**Screenshots**
If applicable, add screenshots.

**Code Example**
```typescript
// Minimal reproduction code
```

**Additional Context**
Any other relevant information.
```

## üìù FAQ

### Common Questions

**Q: Which browsers are supported?**
A: Chrome 113+, Firefox 110+, Safari 16.4+ (with WebGPU). Falls back to WebGL/Canvas2D for older browsers.

**Q: Can I use Tessera in production?**
A: Tessera is currently in planning/development phase. Check the project status in README.md.

**Q: How do I add custom annotation types?**
A: See the [Plugin System documentation](docs/10-plugin-system.md).

**Q: Does Tessera support DICOM?**
A: Yes, DICOM support is planned. See [formats documentation](docs/02-tech-stack.md).

**Q: How do I contribute?**
A: See [CONTRIBUTING.md](CONTRIBUTING.md) for detailed guidelines.
